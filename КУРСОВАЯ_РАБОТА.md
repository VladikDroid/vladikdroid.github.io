# ВВЕДЕНИЕ

## Актуальность темы

Молодёжный сленг представляет собой уникальный лингвистический феномен, отражающий культурные, социальные и исторические изменения в обществе. На протяжении XX–XXI веков русский молодёжный сленг претерпел значительную эволюцию: от уголовного жаргона 1920-х годов до интернет-мемов 2020-х.

Актуальность данного исследования обусловлена следующими факторами:

1. **Лингвистическая ценность**. Сленг является живым свидетельством языковых изменений и требует систематизации для научных исследований.

2. **Культурное наследие**. Многие сленговые выражения уходят из активного употребления, и их фиксация важна для сохранения культурной памяти.

3. **Образовательный потенциал**. Структурированная база данных сленга может использоваться в образовательных целях при изучении истории языка, социологии и культурологии.

4. **Технологический аспект**. Современные веб-технологии позволяют создать доступный инструмент для исследования и популяризации знаний о сленге.

## Степень научной разработанности проблемы

Проблематикой изучения сленга занимались многие исследователи:

- **Лингвистические исследования**: В.М. Мокиенко, В.И. Новиков, А.М. Земсков
- **Социолингвистика**: М.А. Кронгауз, Е.В. Иванова
- **Компьютерная лингвистика**: проекты Urban Dictionary, Lurkmore

Однако комплексных веб-приложений, охватывающих весь период с 1920-х по 2020-е годы с научной систематизацией, на момент написания работы не существовало.

## Цель работы

Разработка полнофункционального веб-приложения для систематизации, хранения и исследования русского молодёжного сленга XX–XXI веков.

## 1.3. Реляционные базы данных

### 1.3.1. Обоснование выбора реляционной модели

Для данного проекта была выбрана реляционная модель базы данных по следующим причинам:

1. **Структурированность данных**. Сленговые слова имеют чёткую структуру: значение, период, категория, теги.

2. **Связи между сущностями**. Необходимость поддерживать связи:
   - Один-ко-многим (период → слова)
   - Многие-ко-многим (слова ↔ теги)

3. **Целостность данных**. Реляционные БД обеспечивают:
   - Ссылочную целостность (foreign keys)
   - Проверку типов данных
   - ACID-транзакции

4. **Зрелость технологий**. SQLAlchemy предоставляет:
   - ORM для работы с объектами
   - Миграции схемы
   - Пул соединений

### 1.3.2. Преимущества SQLite для проекта

| Критерий | SQLite | PostgreSQL | MySQL |
|----------|--------|------------|-------|
| **Простота развёртывания** | ✅ Файл БД | ❌ Сервер | ❌ Сервер |
| **Производительность (чтение)** | ✅ Высокая | ✅ Высокая | ✅ Высокая |
| **Производительность (запись)** | ⚠️ Средняя | ✅ Высокая | ✅ Высокая |
| **Требования к ресурсам** | ✅ Минимальные | ⚠️ Средние | ⚠️ Средние |
| **Поддержка транзакций** | ✅ Да | ✅ Да | ✅ Да |
| **Масштабируемость** | ⚠️ Ограниченная | ✅ Высокая | ✅ Высокая |

**Вывод**: SQLite выбран для курсового проекта из-за простоты развёртывания и достаточной производительности для демонстрационных целей.

### 1.4.2. Описание связей

## 1.6. Нормализация БД

### 1.6.1. Первая нормальная форма (1NF)

**Требование**: Все атрибуты должны быть атомарными (неделимыми).

**Проблема до нормализации**:
```
Word(word, meaning, period, tags)
где tags = ['интернет', 'мемы', '2020-е']
```

**Решение**: Вынести теги в отдельную таблицу `tags` и связующую таблицу `word_tags`.

```sql
-- Таблица words (атомарные поля)
CREATE TABLE words (
    id INTEGER PRIMARY KEY,
    word VARCHAR(100),
    meaning TEXT,
    period_id INTEGER
);

-- Таблица tags
CREATE TABLE tags (
    id INTEGER PRIMARY KEY,
    name VARCHAR(50) UNIQUE
);

-- Связующая таблица
CREATE TABLE word_tags (
    word_id INTEGER,
    tag_id INTEGER,
    PRIMARY KEY (word_id, tag_id)
);
```

**Результат**: ✅ 1NF достигнута

### 1.6.2. Вторая нормальная форма (2NF)

**Требование**: Таблица должна быть в 1NF, и все неключевые атрибуты должны полностью зависеть от первичного ключа.

**Проблема до нормализации**:
```
Word(id, word, meaning, period_name, period_decade, period_color)
```
Здесь `period_name`, `period_decade`, `period_color` зависят от `period_id`, а не от `id` слова.

**Решение**: Вынести данные о периодах в отдельную таблицу.

```sql
CREATE TABLE periods (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100),
    decade VARCHAR(20),
    color VARCHAR(20)
);

-- В таблице words оставляем только ссылку
CREATE TABLE words (
    id INTEGER PRIMARY KEY,
    word VARCHAR(100),
    meaning TEXT,
    period_id INTEGER REFERENCES periods(id)
);
```

**Результат**: ✅ 2NF достигнута

### 1.6.3. Третья нормальная форма (3NF)

**Требование**: Таблица должна быть в 2NF, и не должно быть транзитивных зависимостей.

**Проблема до нормализации**:
```
Word(id, word, meaning, category_id, category_name, category_description)
```
`category_name` и `category_description` транзитивно зависят от `category_id`.

**Решение**: Вынести категории в отдельную таблицу.

```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100),
    description TEXT,
    color VARCHAR(20)
);

-- В таблице words только ссылка
CREATE TABLE words (
    id INTEGER PRIMARY KEY,
    word VARCHAR(100),
    meaning TEXT,
    category_id INTEGER REFERENCES categories(id)
);
```

**Результат**: ✅ 3NF достигнута

### 1.6.4. Итоговая нормализованная схема

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   periods   │     │   words     │     │ categories  │
├─────────────┤     ├─────────────┤     ├─────────────┤
│ id (PK)     │────<│ period_id   │     │ id (PK)     │
│ name        │ 1:N │ category_id │>────│ name        │
│ decade      │     │ word        │     │ description │
│ color       │     │ meaning     │     │ color       │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           │
                    ┌──────┴──────┐
                    │             │
                    ▼             ▼
             ┌─────────────┐ ┌─────────────┐
             │   users     │ │   tags      │
             ├─────────────┤ ├─────────────┤
             │ id (PK)     │ │ id (PK)     │
             │ username    │ │ name        │
             │ email       │ │ color       │
             │ password    │ └──────┬──────┘
             └─────────────┘        │
                                    │ N:M
                                    │
                             ┌──────┴──────┐
                             │  word_tags  │
                             ├─────────────┤
                             │ word_id (PK)│
                             │ tag_id (PK) │
                             └─────────────┘
```

**Вывод**: База данных приведена к 3NF, что обеспечивает:
- ✅ Отсутствие избыточности данных
- ✅ Минимизацию аномалий вставки/обновления/удаления
- ✅ Целостность данных

@app.route('/search')
def search():
    query = request.args.get('q', '')
    period_id = request.args.get('period', type=int)
    category_id = request.args.get('category', type=int)
    sort_by = request.args.get('sort', 'word')
    page = request.args.get('page', 1, type=int)
    
    words_query = Word.query
    
    # Фильтрация
    if query:
        words_query = words_query.filter(
            db.or_(
                Word.word.ilike(f'%{query}%'),
                Word.meaning.ilike(f'%{query}%'),
                Word.synonyms.ilike(f'%{query}%')
            )
        )
    
    if period_id:
        words_query = words_query.filter(Word.period_id == period_id)
    
    if category_id:
        words_query = words_query.filter(Word.category_id == category_id)
    
    # Сортировка
    if sort_by == 'word':
        words_query = words_query.order_by(Word.word)
    elif sort_by == 'popularity':
        words_query = words_query.order_by(desc(Word.popularity))
    elif sort_by == 'created':
        words_query = words_query.order_by(desc(Word.created_at))
    
    # Пагинация (12 слов на страницу)
    pagination = words_query.paginate(page=page, per_page=12, error_out=False)
    
    return render_template('search.html',
                         words=pagination.items,
                         pagination=pagination,
                         query=query,
                         periods=Period.query.all(),
                         categories=Category.query.all())
```

def validate_word_data(data):
    errors = []
    
    # Проверка обязательных полей
    if not data.get('word', '').strip():
        errors.append('Слово обязательно')
    
    if not data.get('meaning', '').strip():
        errors.append('Значение обязательно')
    
    # Проверка длины
    if len(data.get('word', '')) > 100:
        errors.append('Слово слишком длинное (макс. 100 символов)')
    
    if len(data.get('meaning', '')) > 2000:
        errors.append('Значение слишком длинное (макс. 2000 символов)')
    
    # Проверка периода
    period_id = data.get('period_id', type=int)
    if period_id and not Period.query.get(period_id):
        errors.append('Неверный период')
    
    return errors
```

def process_tags(word, tags_str):
    """Обработка строки тегов и присвоение слову."""
    word.tags = []
    
    if tags_str:
        tag_names = [t.strip() for t in tags_str.split(',')]
        for tag_name in tag_names:
            if tag_name:
                tag = Tag.query.filter_by(name=tag_name).first()
                if not tag:
                    tag = Tag(name=tag_name)
                    db.session.add(tag)
                word.tags.append(tag)
    
    db.session.commit()

    query = request.args.get('q', '')
    period_id = request.args.get('period', type=int)
    category_id = request.args.get('category', type=int)
    sort_by = request.args.get('sort', 'word')

    words_query = Word.query
    
    if query:
        words_query = words_query.filter(
            db.or_(
                Word.word.ilike(f'%{query}%'),
                Word.meaning.ilike(f'%{query}%'),
                Word.synonyms.ilike(f'%{query}%')
            )
        )
    
    if period_id:
        words_query = words_query.filter(Word.period_id == period_id)

        words_query = words_query.filter(Word.category_id == category_id)
    
    if sort_by == 'word':
        words_query = words_query.order_by(Word.word)
    elif sort_by == 'popularity':
        words_query = words_query.order_by(desc(Word.popularity))
    elif sort_by == 'created':
        words_query = words_query.order_by(desc(Word.created_at))
    
    pagination = words_query.paginate(page=page, per_page=12, error_out=False)

from flask_sqlalchemy import SQLAlchemy
from flask_login import UserMixin
from werkzeug.security import generate_password_hash, check_password_hash

db = SQLAlchemy()

word_tags = db.Table('word_tags',

    db.Column('tag_id', db.Integer, db.ForeignKey('tags.id'), primary_key=True)
)

class User(UserMixin, db.Model):
    __tablename__ = 'users'
    
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)
    password_hash = db.Column(db.String(256), nullable=False)
    role = db.Column(db.String(20), default='user')
    
    def set_password(self, password):
        self.password_hash = generate_password_hash(password)
    
    def check_password(self, password):
        return check_password_hash(self.password_hash, password)

class Word(db.Model):
    __tablename__ = 'words'
    
    id = db.Column(db.Integer, primary_key=True)
    word = db.Column(db.String(100), nullable=False, index=True)
    meaning = db.Column(db.Text, nullable=False)
    period_id = db.Column(db.Integer, db.ForeignKey('periods.id'))
    category_id = db.Column(db.Integer, db.ForeignKey('categories.id'))
    author_id = db.Column(db.Integer, db.ForeignKey('users.id'))
    origin = db.Column(db.String(200))
    example = db.Column(db.Text)
    synonyms = db.Column(db.Text)
    antonyms = db.Column(db.Text)
    popularity = db.Column(db.Integer, default=0)
    is_verified = db.Column(db.Boolean, default=False)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    
    tags = db.relationship('Tag', secondary=word_tags, backref='words')
    
    def get_synonyms_list(self):
        return [s.strip() for s in self.synonyms.split(',')] if self.synonyms else []
    
    def get_vote_score(self):
        return sum(v.vote_type for v in self.votes)

class Period(db.Model):
